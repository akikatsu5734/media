# アキカツ AI活用戦略：課題解決プロダクトへの進化

## 背景
- 空き家所有者向けの情報メディア「アキカツ」を運営中
- 現状は記事量産フェーズだが、真の目的は空き家オーナーの課題解決

## 現状課題
- 記事数の達成が目的化しており、読まれない記事が多い
- 地域情報コンテンツが薄く、地域メディアとして機能不全
- 記事制作・分析・改善がほぼ人力で非効率

## 目指す状態
- 「記事を読んで終わり」ではなく、課題解決まで導くプロダクトへ進化
- AIが記事生成・分析・改善・相談まで担う内製ワークフローを確立

## 重要な評価軸
- 質（Quality）：読まれる・価値を生む記事・機能か
- 量（Scale）：少ない人手でスケールできるか
- スピード（Speed）：素早く試して検証できるか
- 最小コスト最大価値（Capital Efficiency）：数千〜数万円で始められるか

## 主要戦略
- 戦略1：AI内製ワークフローで記事制作を自動化（速度×品質の両立）
- 戦略2：地域情報コンテンツをAIで拡充（地域メディア価値の確立）
- 戦略3：AI問診・AI相談でユーザー課題解決を直接支援
- 戦略4：GA4/Search Console連携で分析→改善を自動化
- 戦略5：診断・レコメンド・導線最適化でプロダクト化

## 90日アクションプラン
- Day 1-30：AI記事ワークフロー構築・地域ページMVP・分析基盤整備
- Day 31-60：AI問診フロー実装・レコメンド試験導入・改善ループ稼働
- Day 61-90：AI相談エージェントMVP・プロダクト機能拡張・KPI検証

## Claude Code活用方針
- WordPress REST API連携スクリプトの実装
- GA4/Search Console分析→記事改善提案の自動化
- MCPサーバー経由でのWordPress記事投稿ワークフロー
- AI問診フロー・診断UIのプロトタイプ実装

## 優先順位
- 最優先：読まれる記事の分析と改善（即効性が高い）
- 次点：地域情報コンテンツのAI生成・投稿自動化
- 中期：AI問診・診断フローのMVP
- 後期：AI相談エージェント・フルプロダクト化

## すぐやること
- GA4データを取得し「読まれている記事」「離脱が多い記事」を特定
- WordPressのREST APIキーを取得し、Claude Codeからの投稿テストを実施
- 地域ページ1つ（例：世田谷）を対象にAI生成コンテンツのA/Bテスト

## 後でやること
- AI相談エージェントの本格実装（ユーザー認証・会話ログ管理が必要）
- パーソナライズレコメンドエンジン（データ蓄積後に着手）
- LINEチャット連携（外部API契約・開発工数が必要）
