## 1. 背景

- 事業名：アキカツ（空き家解決情報メディア） https://aki-katsu.co.jp/magazine/
- ターゲット：空き家所有者・空き家所有予備軍
- 現状の運営形態：WordPress + インターン + ChatGPTで記事を量産中
- MVP目標として150記事を計画、残り107記事
- 地域ごとのページ構成あり（例: /magazine/setagaya）
- 課題の本質は「記事が足りない」ではなく「ユーザーの課題が解決されていない」こと

## 3. 目指す状態

**短期（90日後）**
- AI内製ワークフローが稼働し、記事品質と制作速度が両立している
- 地域ページ（最低5地域）の情報量が実用レベルに達している
- GA4データを基にした改善サイクルが週次で回っている

**中期（6ヶ月後）**
- AI問診・診断フローのMVPが公開されており、ユーザーが自分の状況を入力して次のアクションを得られる
- 読者を「記事を読む人」から「課題を解決できる人」に変換できている

**長期（1年後）**
- アキカツが「空き家の困りごとを解決するプロダクト」として認知される
- AI相談エージェントが空き家所有者のファーストコンタクトを担っている

## 4. 評価軸（意思決定の基準）

施策を判断するときは必ずこの4軸で評価する。

| 軸 | 問い |
|----|------|
| **質 (Quality)** | 読まれるか、ユーザーの課題を解決できるか |
| **量 (Scale)** | 少ない人手でスケールできるか |
| **スピード (Speed)** | 1〜2週間以内に試せるか |
| **資本効率 (Capital Efficiency)** | 月額数千〜数万円で始められるか |

> 4軸のうち2つ以上で「No」なら着手しない。

## 5. 戦略の柱

### 柱1：分析→改善ループをAIで自動化する（最優先）
読まれている記事と読まれていない記事を特定し、改善提案→再制作のサイクルをAIで回す。  
記事を増やす前に、今ある記事を最大化する。

### 柱2：地域情報コンテンツをAIで拡充する（優先）
地域ごとの空き家件数・地価・住宅事情・行政支援情報をAIで収集・整形し、  
地域ページの情報密度を上げる。地域名での検索流入を増やす。

### 柱3：AI内製記事ワークフローを構築する（優先）
WordPress REST API + Claude Code + MCPで、  
キーワード選定→記事生成→校正→投稿までを自動化する。  
インターンの作業を「確認・編集」に集中させる。

### 柱4：ユーザー課題解決の導線を設計する（中期）
記事を読んだ後に「あなたの状況に合った次のステップ」へ誘導する。  
問診フロー・診断UI・関連記事レコメンドを段階的に実装する。

### 柱5：AI問診・AI相談のMVPを作る（中期）
空き家所有者の状況（所有形態・悩み・地域・予算）を入力させ、  
最適なアドバイスと関連記事・専門家への送客を提案するフローを実装する。

## 9. Claude Codeの具体的な役割

### できること（積極的に使う）

| 役割 | 具体的な使い方 |
|------|--------------|
| **分析スクリプト** | GA4/Search Console APIからデータ取得し、記事パフォーマンスを自動集計 |
| **改善提案** | 記事URLを入力すると、SEO・構成・CTA改善点を出力するスクリプト |
| **記事生成** | キーワード・ターゲット・構成指示を与えて記事本文を生成 |
| **WordPress連携** | REST API経由で下書き作成・タグ付け・カテゴリ設定まで自動化 |
| **地域情報収集** | 地域名を入力すると、行政情報・空き家件数・地価などを収集・整形 |
| **MCPサーバー** | WordPressへの投稿・更新・記事一覧取得をMCPプロトコル経由で統一管理 |
| **問診フロー実装** | JavaScriptベースの問診コンポーネントをWordPressに組み込む |
| **内部リンク提案** | 記事一覧を読み込み、関連記事のリンクを自動でサジェスト |
| **プロンプト管理** | 用途別プロンプトをテンプレートとして管理・バージョン管理 |

### 向いていないこと（別手段で補う）

| 役割 | 理由 | 代替手段 |
|------|------|---------|
| リアルタイムUI/UX改善 | ブラウザ確認が必要 | 人間がレビュー |
| 事実確認・一次情報収集 | ハルシネーションリスク | 公的機関サイトを参照 |
| 画像生成・デザイン | 専門ツールに劣る | Canva / Midjourney |
| ユーザー感情の解釈 | データが必要 | ヒートマップ・録画ツール |
