## フェーズ1：Day 1-30（分析基盤 × 記事ワークフロー × 地域情報MVP）

---

### TASK-001：GA4データ取得スクリプトの実装

| 項目 | 内容 |
|------|------|
| 目的 | 記事ごとのセッション数・直帰率・滞在時間を自動取得し、改善対象を特定できるようにする |
| 優先度 | **高** |
| 期限 | Day 1-30 |
| 担当 | Claude Code |
| 依存関係 | GA4アカウントへのAPIアクセス権限（サービスアカウント設定が必要） |
| 完了条件 | スクリプト実行でCSVまたはMarkdownの記事パフォーマンスレポートが出力される |
| 備考 | Google Analytics Data API v1 を使用。認証はサービスアカウントJSONで行う |

---

### TASK-002：Search Console データ取得スクリプトの実装

| 項目 | 内容 |
|------|------|
| 目的 | 検索クエリ・表示回数・クリック率・掲載順位を取得し、キーワード機会を特定する |
| 優先度 | **高** |
| 期限 | Day 1-30 |
| 担当 | Claude Code |
| 依存関係 | TASK-001と同じGoogleサービスアカウント設定 |
| 完了条件 | URLごとの上位クエリと順位一覧が出力される |
| 備考 | Google Search Console API v1 を使用 |

---

### TASK-003：記事パフォーマンスレポートの作成

| 項目 | 内容 |
|------|------|
| 目的 | 読まれている記事トップ20・改善余地が大きい記事20本を特定する |
| 優先度 | **高** |
| 期限 | Day 1-30 |
| 担当 | 両方 |
| 依存関係 | TASK-001、TASK-002 |
| 完了条件 | 「読まれている記事」「読まれていない記事」「順位が落ちている記事」が一覧化されている |
| 備考 | Claude Codeで集計・Markdownで出力し、人間が最終判断する |

---

### TASK-004：WordPress REST API 接続テスト

| 項目 | 内容 |

## 保留・今はやらないタスク

| タスク名 | 保留理由 | 再検討タイミング |
|---------|---------|----------------|
| LINE公式アカウント連携 | 外部API契約・運用コストが発生する | 問診フローMVP結果確認後 |
| パーソナライズレコメンド | ログイン基盤・データ蓄積が必要 | 問診ログ100件超え後 |
| AI相談チャット（本格版） | 認証・会話ログ管理が必要 | Day 90以降 |
| 独自AIモデルのファインチューニング | コスト・工数に対しROIが低い | 2年後以降 |
| スマホアプリ開発 | 開発コスト大・Web優先で十分 | ユーザー数安定後 |
